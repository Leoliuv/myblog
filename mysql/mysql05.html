# MySQL5.7升级

# 1. 升级方法
  • 原地升级，直接替换老的MySQL程序
    1. 停止老MySQL
    2. 使用新的MySQL程序替换替换老的MySQL程序
    3. 重启MySQL
    4. 运行mysql_upgrade
  • 逻辑升级，导出以前数据再导入新版本
    1. 从老版本中导入mysql
    2. 安装新版本
    3. 将数据导入到新版mysql
    4. 运行mysql_upgrade

# 2.升级路径

  • 5.7 发布的系列版本之间升级。可以从5.7.9升级到5.7.10，跳过小版本升级也是可以的，从5.7.9升级到5.7.11
  • 发行版的大版本升级。从5.6.*升级到5.7。升级跨域不止一个版本也是支持的，但是一次只能升级一个发行版，例如现在运行的是5.5，要升级到5.7就需要先升级到5.6再升级到5.7。

# 3.升级前的准备

  1. 备份一份当前的数据库

  2. 查看要升级版本的新特性

  3. 确实在升级前和升级后需要进行操作动作

  4. 是否需要改变表格式、字符集、collections等。

  5. 登入要升级的数据库查看数据库的版本

    select version()

    status

    1. 下载要升级的安装包，上传服务器
    2. 关闭分布式数据库代理，防止升级期间连接数据库

# 升级步骤

  1. 备份数据库

    ./bin/mysqldump -uroot -proot --socket=/paas/iddbs/liuli/mysql01/mysql/21000/mysql.sock --single-transaction --skip-add-locks --master-data=2 --all-databases>/paas/iddbs/liuli/mysql01/mysql/21000back/1010_57_27.sql

  2. 将数据库停止

    ./bin/mysqladmin -uroot -proot -h127.0.0.1 -P21000 shutdown

  3. 备份就得MySQL

    mv mysql mysql_20191008

    将准备好的新版本解压后移动到原来mysql的目录下，之后移动MySQL的数据目录和配置文件etc的目录

  4. 启动并登录MySQL

    ./bin/mysqld_safe --defaults-file=/paas/iddbs/liuli/mysql01/mysql/etc/my21000.cnf &

    ./bin/mysql -uroot -proot -h127.0.0.1 -P21000

  5. 升级操作

    ./bin/mysql_upgrade -uroot -proot -h127.0.0.1 -P21000

  6. 升级之后重启MySQL

  7. 修改配置文件

    在~/.bash_profile增加配置

    export PATH=/paas/iddbs/mysql/bin:$PATH 然后刷新配置文件 source ~/.b ash_profile

    补充：在升级的时候，一般先对从库进行升级，若升级失败也不会影响到生产，如果升级成功，测试也成功，就慢慢的将其他的从库升级到新版本，最后将主库下线，提升一个新的从库作为新的主库，对旧的主库进行升级

# MySQL从5.7升级到8.0版本

按照上述：

  1. 备份数据库
  2. 将原来数据库进行改名或
  3. 将新的安装文件改为原名
  4. 将配置数据文件和配置文件放入数据库
  5. 启动服务，尝试登入数据库
  6. 运行./bin/mysql_upgrade -uroot -proot -h127.0.0.1 -P21000
  7. 登入数据库查看目前数据库的版本
