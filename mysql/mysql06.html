# percona-toolkit工具包安装使用

# 1. 简介

​ 用来执行各种通过手工执行非常复杂和麻烦的mysql和系统任务，可以查看当前服务的摘要信息，磁盘检测，分析慢查询日志，查找重复索引，实现表同步等等

# 2. 安装

  1. 下载linux通用版本：https://www.percona.com/downloads/percona-toolkit

  2. 解压工具： tar -xzvf percona-toolkit-3.0.13_i386.tar.gz

  3. 检查安装是否成功：

    [iddbs@host-172-18-231-117 percona-toolkit-3.0.9]$ bin/pt-table-checksum --version

    返回版本号则表示成功

# 3. 使用
# 1. 开发类

  1. pt-duplicate-key-checker 检查是否有重复的索引，给出重复的索引信息，包括索引定义，列的数据类型、以及修复建议。该工具以库为最小统计级别

    pt-duplicate-key-checker --host=172.18.231.117 --port=21000 --user='mysql' --password=mysql --databases=test

  2. pt-online-schema-change

    pt-online-schema-change h=172.18.231.117,P=21000,u=mysql,p=mysql,D=test,t=test01 --set-vars --lock-wait-timeout=10 --execute --alter "modify name varchar(15);"

    将test.test01中的name属性修改为varchar(15)

    1. pt-show-grants

    指定具体mysql，打印改mysql的所有授权语句

    pt-show-grants h=172.231.117,P=21000,u=mysql,p=mysql

  3. pt-upgrade

    用来测试新老版本数据库对sql的结果是否一致。两台开发测试数据库，一台是老版本，一台是新版本，来做比较，生产库别用

    vi a.sql
select * from test01 where id =1;

pt-upgrade h=172.18.231.117,P=21000,u=mysql,p=mysql h=172.18.231.117,P=22000,u=mysql,p=mysql --database=test a.sql

# 2.性能类

  1. pt-index-usage

    分析日志中索引使用情况，并出报告

    pt-index-usage /paas/iddbs/liuli/mysql01/mysql/21000/error.log --host=172.18.231.117 --port=21000 --user=mysql --password=mysq

2.pt-pmp

​ 为查询结果跟踪，并汇总跟踪结果.一是获取进程的堆栈信息，二是对这些堆栈信息进行汇总。底层程序行为效率跟踪

  1. pt-visual-explain

    格式化执行计划，将explain的结果按照 tree 方式输出

  2. pt-table-usage

    简单分析下语句结构

    pt-table-usage h=172.18.231.117,P=21000,u=mysql,p=mysql,D=test --query="select * from test01 where id=2;"

# 3.配置类

  1. pt-config-diff

    检查来自于通过命令行指定的两个或者两个以上的config中的服务器系统变量值，以此来diff不同MySQL实例间的配置。

    pt-config-diff h=172.18.231.117,P=21000,u=mysql,p=mysql h=172.18.231.117,P=22000,u=mysql,p=mysql

  2. pt-mysql-summary

    对mysql配置和status进行汇总，主要用来输出MySQL的基本信息，可以作为数据库巡检以及刚开始熟悉数据库环境时候进行使用

    pt-mysql-summary --host=172.18.231.117 --port=21000 --user=mysql --password=mysql

  3. pt-variable-advisor

    是一款分析参数，并且给出参数设置建议的一款PT工具，得出的结果仅供参考，可根据业务需求做变动

    pt-variable-advisor u=mysql,p=mysql,h=172.18.231.117,P=21000

# 4.监控类（未完）

  1. pt-mext

    并行查看status样本信息

    pt-mext -r -- mysqladmin ext -i10 -c3 --host=172.18.231.117 --port=21000 --user=mysql --password=mysql

  2. pt-query-digest

    pt-query-digest 172.18.231.117-21000-slowsql-error.log
# 5.复制类（未完）

  1. pt-heartbeat

    a. 创建heartbeat表

    b. 更新heartbeat表，获得最新的数据

    c. 开启监控进程，这个是不停的刷新的

  2. pt-slave-delay

    故意让主上的操作延迟制定的时间写入到从，解决主库误操作的问题，但是需要长期运行该命令，比较耗费资源

  3. pt-slave-find

    查找和打印MySQL从库的复制层次树

  4. pt-slave-restart

    监控一个或者多个MySQL复制slave的错误，然后当复制停止时试图重启

  5. pt-table-checksum

    在线检测主从数据一致性的工具。通过pt-table-checksum检测出数据不一致后，可以通过pt-table-sync进行数据不一致的修复

  6. pt-table-sync
